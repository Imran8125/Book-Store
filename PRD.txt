BOOK STORE PROJECT PRD

Project Name
Book Store / BookEase

1. Project Overview
The Book Store project is an online platform that allows users to browse, search, and purchase books easily. The system is designed to provide a smooth shopping experience for customers while also supporting sellers and administrators in managing books, inventory, orders, and user accounts.

2. Project Goals
- Provide a user-friendly online bookstore experience.
- Allow users to register, log in, and manage their profiles securely.
- Enable users to browse books by category, genre, author, or popularity.
- Support a complete purchase workflow from selection to order confirmation.
- Allow sellers to add and manage book listings.
- Enable admins to manage users, sellers, books, and orders efficiently.

3. Target Users
- Customer/User: Browses books, places orders, and views order history.
- Seller: Adds books, manages inventory, and fulfills orders.
- Admin: Oversees the system, manages users, sellers, books, and orders.

4. Functional Requirements
4.1 User Features
- User registration and authentication
- Secure login and logout
- Profile management
- Book browsing and search
- Filter books by genre, author, ratings, or popularity
- View detailed book information including title, author, genre, description, price, and availability
- Add books to cart and specify quantities
- Choose book formats such as e-book, paperback, or special edition
- Complete secure checkout and place orders
- Receive order confirmation with order ID, price, and book details
- View current and past orders and track delivery status

4.2 Seller Features
- Seller registration and profile management
- Add new books with complete information
- Update existing book listings
- Manage book quantity and availability
- Process and ship orders

4.3 Admin Features
- Manage user accounts
- Manage seller accounts and approvals
- Add, update, or remove book listings
- Manage inventory and stock levels
- Monitor orders and platform activities
- Manage user and seller ratings

5. Technical Architecture
The application is designed with a layered and scalable architecture:
- User Interface (UI): Provides the front-end experience for browsing books and managing accounts.
- Web Server: Hosts the application and delivers dynamic pages efficiently.
- API Gateway: Routes client requests to the correct backend services.
- Authentication Service: Handles secure login, registration, and authorization.
- Database: Stores books, users, orders, reviews, inventory, and related records.
- View Books: Allows users to explore the complete catalog.
- Category Selection: Enables browsing by genre and category.
- Order Management Service: Handles cart operations, purchase flow, and order tracking.

6. Entity Relationship Design
The system uses relational database design with the following relationships:
- User-Book Relationship: Many-to-Many (M:M)
  - Implemented using an intermediate entity called Interaction.
  - Stores information such as reading progress, reviews, or ratings.
- Book-Inventory Relationship: One-to-Many (1:M)
  - Each book can have multiple inventory entries, but each inventory item belongs to one book.
- User-Order Relationship: One-to-Many (1:M)
  - A single user can place many orders, and each order belongs to one user.
- Book-Author Relationship: Many-to-Many (M:M)
  - Implemented using a WrittenBy table.
- Book-Genre Relationship: Many-to-Many (M:M)
  - Implemented using a CategorizedAs table.
- Review-User Relationship: Many-to-One (M:1)
  - Each review is written by one user, and a user can write many reviews.

7. User Flow
1. User opens the application.
2. User views the home page with featured books and categories.
3. User logs in or opens their profile.
4. User browses or searches for books.
5. User selects a book and views details.
6. User adds the book to the cart and specifies quantity or format.
7. User reviews the cart and proceeds to secure payment.
8. User receives an order confirmation.
9. User views current and past orders.
10. The process ends once the user completes the intended action.

8. Backend Architecture Pattern
The backend follows the Model-View-Controller (MVC) pattern:
- Model Layer: Handles data schemas and database operations using Mongoose for MongoDB.
- Controller Layer: Receives requests, validates input, calls business logic, and returns responses.
- View Layer / Routing Layer: Defines API endpoints for handling HTTP requests such as GET, POST, PUT, and DELETE.

Benefits of this structure:
- Separation of concerns
- Easier maintenance
- Better scalability
- Reusability of logic
- Improved collaboration between developers

9. Project Setup and Configuration
The project is organized into two main folders:
- Client: Front-end application
- Server: Back-end application

Client setup:
- Create the main project folder.
- Inside it, create Client and Server folders.
- Open the project in VS Code.
- Run: npm create vite@latest . -- --template react
- Select React and JavaScript.
- Run: npm install
- Start the development server: npm run dev

Server setup:
- Open the Server folder in the terminal.
- Run: npm init -y
- Create files such as server.js
- Create folders such as models, controllers, and routes

10. Summary
The Book Store project is a full-stack online bookstore application that supports customers, sellers, and administrators. It provides essential e-commerce and content-management features while following a structured architecture, clear roles, and a scalable backend design.
